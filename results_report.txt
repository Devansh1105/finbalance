================================================================================
  FINDING 2 -- Retained Earnings Is THE Diagnostic Account (Headline Finding)
================================================================================

  Account Difficulty Index (aggregated across all 11 model x strategy runs):

    Rank  Account                     Omit%    Arith%   Correct%   DiffScore
    1     Retained Earnings           72.5%    11.4%    16.1%      0.481
    2     Owner's Equity               2.7%    51.4%    45.9%      0.222
    3     Accumulated Depreciation    28.0%     3.9%    68.1%      0.184
    4     Cash                         0.1%    26.1%    73.8%      0.105
    5     Allowance for Doubtful Acc   8.8%     3.8%    87.4%      0.068

  The #1 and #2 hardest accounts are BOTH equity accounts related to
  closing entries (difficulty score 0.481 and 0.222 respectively).

  RE behaviour by model (omission rate -> CECR):
    GPT-5.2 ZS:  92.8% omit, CECR=8%   (almost never includes RE)
    GPT-5.2 CoT: 100%  omit, CECR=1%   (CoT makes it WORSE)
    Qwen ZS:     93.9% omit, CECR=3%   (same pattern as GPT-5.2)
    Qwen CoT:    93.9% omit, CECR=7%
    Llama:       100%  omit, CECR=1%   (never includes RE)
    Gemini ZS:   10.1% omit, 75.8% arith, CECR=15% (includes but wrong)
    Gemini CoT:  69.7% omit, 14.1% arith, CECR=17%
    gpt-oss ZS:  37.8% omit, CECR=51%  (frequently includes RE)
    gpt-oss CoT: 52.6% omit, CECR=43%  (still includes, but less)

  The CECR Spectrum (1% -> 51%):
    Llama=1%, GPT-5.2=1-8%, DeepSeek ZS=2%, Qwen=3-7%, Gemini=15-17%,
    DeepSeek CoT=23%, gpt-oss=43-51%

  This reveals a closing-entry compliance axis that separates models:
    - "Blindness" (GPT-5.2, Qwen, Llama): omit RE, inflate Owner's Equity
    - "Partial awareness" (Gemini): includes RE but can't compute it
    - "Over-compliance" (gpt-oss): frequently includes RE, often correctly

  PAPER IMPLICATION:
  This is the headline finding. The Account Difficulty Index gives it
  quantitative teeth: RE at 72.5% omission is nearly 2x harder than the
  next account. The CECR spectrum (1-51%) is a novel diagnostic axis --
  models don't uniformly fail at closing entries; they fail in
  characteristically different ways that reveal training biases.

  Quotable: "Retained Earnings is the single hardest account in the
  benchmark (72.5% omission, difficulty score 0.481) -- models either
  omit it entirely, include it with incorrect values, or over-apply
  closing procedures. The CECR spectrum from 1% to 51% reveals five
  qualitatively distinct approaches to the same GAAP requirement."

  Use in: Section 1 Introduction (hook), Section 4.2 Account Difficulty,
  Section 5 Discussion.

  FINDING 3 -- mixed_funding Is the Universal Killer
================================================================================

  Complexity Factor Attribution (DFBS, negative = harder):

  Factor             GPT-5.2  gpt-oss  Qwen    Gemini   Llama
                     ZS/CoT   ZS/CoT   ZS/CoT  ZS/CoT   ZS
  ---------------------------------------------------------------
  mixed_funding      -0.9/+1  -27/-15  -15/-7  -13/-9   -10
  derived_interest   -0.2/+1  -15/-10  -13/-4  -10/-4   -8
  prepaid            -0.0/+2  -15/-6   -5/-2   -9/-9    -11
  debt               -0.6/+3  -7/-6    -4/+0   -13/-5   -2
  depreciable_asset  +6.7/+1  -10/-2   -4/-1   -12/-6   +9
  credit_transaction -0.6/+3  -3/-5    -3/+1   -12/-7   -4
  cogs               +0.7/+2  -12/-6   -8/-2   -9/-6    -3

  KEY: GPT-5.2 is IMMUNE to all complexity factors (|DFBS| < 3.4)
  while mixed_funding causes -15 to -27 FBS drops for other models.

  Difficulty-controlled DFBS (stratified by level, removes confounding):
    - After controlling, `prepaid` and `mixed_funding` remain genuinely
      harmful for most models
    - `depreciable_asset` raw effect is confounded by difficulty (only
      appears at higher levels); controlled effect is near zero for most
    - GPT-5.2 immunity persists even after controlling

  PAPER IMPLICATION:
  mixed_funding (split cash/credit purchases requiring 3-entry journal
  entries) is the clearest discriminator of model capability. A model
  that handles mixed_funding correctly has internalized multi-account
  reasoning. This is actionable: practitioners should specifically test
  LLMs on split-allocation transactions before deploying them for
  financial tasks.

  Use in: Section 4.3 Complexity Analysis, Table (CFA heat map).


================================================================================
  FINDING 4 -- CoT Operates Through Different Mechanisms Per Model
================================================================================

  CoT Benefit Decomposition:

  Model          DFBS   Primary Mechanism           Parse+  DOmit  DArith DHalluc
  -------------------------------------------------------------------------------
  DeepSeek       +32.6  Latent knowledge mobilise    0    -0.03   N/A    -0.00
  gpt-oss-120b   +8.2   Parse recovery               16   -0.21  +0.22  +0.40
  Gemini          +4.4   Arithmetic improvement        0   +0.66  -2.09  -0.04
  Qwen            +3.2   L4/L5 difficulty only         1   +0.04  -0.35  +0.14
  GPT-5.2         +0.7   NEUTRAL (p=0.53)              2   +0.10  +0.19  -0.12

  DeepSeek-v3.2 CoT gain (+32.6) is driven by:
    - BA jumps from 15% to 89% (arithmetic engine activates with CoT)
    - ACR drops slightly (96.8% -> 87.4%) -- CoT trades some coverage
      for dramatically better arithmetic
    - CECR increases from 2% to 23% (partial closing-entry activation)
    - By difficulty: CoT gain is huge at L2 (+41 FBS), L3 (+37 FBS)
      where ZS fails completely; consistent gains through L4-L5
    - This is a QUALITATIVELY different mechanism from all other models:
      not parse recovery, not arithmetic precision, but fundamental
      mode switch in how the model processes the task

  gpt-oss-120b CoT gain (+8.2) is driven by:
    - 16 problems go from parse_fail -> success (the largest parse recovery)
    - But hallucinations INCREASE (+0.40/problem) and arithmetic worsens
    - CoT fixes the output format, not the reasoning

  Gemini CoT gain (+4.4) is driven by:
    - Arithmetic errors drop by -2.09/problem (genuine reasoning benefit)
    - But omissions INCREASE (+0.66/problem) -- CoT trades coverage for precision
    - ACR drops from 98% to 92% while BA improves from 73% to 80%

  Qwen CoT gain (+3.2) is concentrated at L4/L5:
    - L1-L3: DFBS = -0.4, +0.2, -0.0 (essentially zero)
    - L4: DFBS = +3.9  (5 improved, 3 degraded)
    - L5: DFBS = +12.4 (12 improved, 6 degraded)
    - CoT is a complexity-threshold intervention: useless below L4, valuable above

  GPT-5.2 CoT gain (+0.7) is NOT significant (p=0.53):
    - 21 improved, 44 degraded, 35 unchanged
    - More problems degrade than improve!
    - ALA actually DECREASES (-0.014)
    - CECR drops from 8% to 1% (CoT reinforces RE omission)

  CoT has highest variance at highest difficulty:
    gpt-oss L4: std=54.9, L5: std=63.5
    -> CoT is a high-risk/high-reward intervention at hard levels

  PAPER IMPLICATION:
  This decomposition directly challenges the "CoT helps reasoning" narrative.
  For accounting generation:
    - CoT helps FORMAT (parse recovery), not REASONING (arithmetic)
    - CoT helps WEAK models more than STRONG ones (inverse scaling)
    - CoT increases variance, especially at high difficulty
    - CoT can REINFORCE incorrect patterns (GPT-5.2 CECR: 8% -> 1%)

  Practitioners should NOT assume CoT improves financial reasoning. Its
  effect is model-specific and operates through different mechanisms.

  Use in: Section 4.4 Prompting Strategy Analysis, Figure (CoT decomp).


================================================================================
  FINDING 5 -- Equity Is the Hardest Section (for Closing-Entry-Aware Models)
================================================================================

  Section-Level Error Asymmetry (mean absolute error $):

  Model                        Assets   Liabilities   Equity    Hardest
  ---------------------------------------------------------------------
  GPT-5.2 CoT                  $0       $0            $0        --
  GPT-5.2 ZS                   $42      $42           $0        Assets
  gpt-oss CoT                  $1,060   $654          $5,562    EQUITY
  gpt-oss ZS                   $2       $36           $2,096    EQUITY
  Qwen CoT                     $1,429   $290          $362      Assets
  Qwen ZS                      $3,721   $415          $2,191    Assets
  Gemini CoT                   $9,931   $105          $5,752    Assets
  Gemini ZS                    $26,100  $11,798       $18,018   Assets
  Llama ZS                     $62,154  $14,221       $60,379   Assets

  Cross-model mean: Assets=$11,604  Liabilities=$3,062  Equity=$10,485

  KEY: gpt-oss-120b is the ONLY model where equity is the hardest section.
  This is because it attempts closing entries (CECR=43-51%) but often
  gets the Retained Earnings value wrong, creating equity-section errors.
  All other models fail hardest on assets (miscounting cash, depreciation).

  Liabilities are universally the easiest section ($3,062 mean). This makes
  sense: liabilities are the simplest to compute (fewer accounts, no
  closing entries, straightforward credit-side accumulation).

  PAPER IMPLICATION:
  Section error asymmetry connects directly to the closing-entry story.
  Models that avoid closing entries (GPT-5.2, Qwen) have equity errors
  near zero (because Owner's Equity absorbs everything). Models that
  attempt closing entries (gpt-oss) shift their errors into equity.
  This reveals a trade-off: attempting GAAP compliance (closing entries)
  introduces equity-section risk that simpler approaches avoid.

  Use in: Section 4.5 Section-Level Analysis, connects to CECR discussion.

================================================================================

  FBS by difficulty level:

  Model                  L1    L2    L3    L4    L5    Deg/Lvl  Cliff     Drop
  ---------------------------------------------------------------------------
  DeepSeek ZS           76.7  52.8  51.8  45.7  37.7   9.7    L1->L2   23.8 !!!
  gpt-oss ZS            80.6  88.8  97.0  66.6  57.6   5.7    L3->L4   30.4
  Qwen ZS               86.2  89.7  92.2  85.4  64.0   5.6    L4->L5   21.5
  Gemini ZS             88.5  79.0  85.0  76.1  66.5   5.5    L4->L5    9.6
  gpt-oss CoT           90.1  91.3  95.0  83.6  71.7   4.6    L4->L5   11.8
  Gemini CoT            89.1  84.3  87.8  81.8  74.4   3.7    L4->L5    7.5
  Llama ZS              36.3  33.8  41.9  31.9  23.3   3.3    L3->L4   10.1
  DeepSeek CoT          88.8  93.5  89.1  79.0  77.5   2.8    L4->L5   11.5
  Qwen CoT              85.8  89.9  92.2  89.3  76.3   2.4    L4->L5   13.0
  GPT-5.2 ZS            90.7  88.8  92.2  92.3  87.0   0.9    L4->L5    5.2
  GPT-5.2 CoT           88.2  91.8  91.2  91.7  91.4  -0.8    --        0.6

  Unique observations:

  1. GPT-5.2 CoT has NEGATIVE degradation (-0.8/level) -- it gets BETTER
     at higher difficulty. This is because L1 RE omission hurts more at
     low complexity (fewer accounts = each omission has larger ALA impact).

  2. gpt-oss ZS has the most dramatic cliff: L3=97.0 -> L4=66.6 (-30.4).
     This L3 score is the SINGLE BEST LEVEL RESULT of any model. Yet
     the model collapses past L3, suggesting a sharp complexity ceiling.

  3. Most models cliff at L4->L5 (mixed_funding, derived_interest appear).
     Llama and gpt-oss ZS cliff at L3->L4 (earlier collapse).

  4. CoT consistently shifts the cliff rightward (later onset):
     gpt-oss: L3->L4 cliff (ZS) becomes L4->L5 (CoT)
     Qwen: no cliff (ZS L4->L5=21.5) becomes smaller (CoT L4->L5=13.0)
     DeepSeek: L1->L2 cliff (ZS, -23.8) disappears entirely under CoT

  5. DeepSeek CoT peaks at L2 (FBS=93.5) -- second-best single-level result
     across all models after gpt-oss ZS L3=97.0. This is notable because
     DeepSeek ZS has already collapsed at L2 (52.8). The CoT intervention
     converts a CLIFF model into a MODERATE-degradation model.

  PAPER IMPLICATION:
  Degradation rate and cliff point are natural benchmark outputs. CV
  classification (ROBUST/MODERATE/STEEP/CLIFF) gives reviewers a single
  number to evaluate model quality. The gpt-oss L3 peak anomaly is a
  compelling case study for non-monotonic complexity response -- the
  model has a "sweet spot" that collapses past a specific complexity
  threshold.

  Use in: Section 4.1 (Figure F1 capability curve), Section 5 Discussion.

================================================================================
  ADDING A NEW MODEL
================================================================================

  When a new model is evaluated, run these steps to update all analyses:

  # 1. Run evaluation
  python scripts/run_evaluation.py --dataset data/test.jsonl \
      --models <model_id> --strategies zero_shot,cot --workers 8 \
      --max-tokens 32768

  # 2. Compute extended metrics (updates leaderboard.json automatically)
  python scripts/compute_extended_metrics.py

  # 3. Failure analysis (per strategy)
  python scripts/analyze_failures.py --results results/<model>_zero_shot.json \
      --dataset data/test.jsonl --output results/failure_analysis_<model>_zero_shot.json
  python scripts/analyze_failures.py --results results/<model>_cot.json \
      --dataset data/test.jsonl --output results/failure_analysis_<model>_cot.json

  # 4. Regenerate deep analysis (CFA, Account Index, CoT Decomp, etc.)
  python scripts/deep_analysis.py

  # 5. Re-run permutation tests (adds new model to significance table)
  python scripts/permutation_tests.py

  # 6. Regenerate all figures (auto-discovers new model results)
  python scripts/generate_figures.py

  All scripts auto-discover result files matching data/test.jsonl problem IDs.
  No code changes needed -- just add the result file and re-run the pipeline.
