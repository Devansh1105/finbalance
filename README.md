# FinBalance — Multi-Document Accounting Reconciliation Benchmark

A benchmark for evaluating LLMs on document-grounded accounting reconciliation.
Each record is a packet of OCR-style accounting documents (invoices, bank
statements, payment notices, schedules, certificates, plus distractors) and a
deterministic ground-truth ledger. Models must produce journal entries with
supporting `doc_refs`, the final balance sheet, and inconsistency flags for
packets that contain a deliberate contradiction.

The benchmark is generated by a deterministic double-entry ledger engine, so
every record's ground truth — entries, balance sheet, opening trial balance —
is computable rather than human-labeled. Eight industries × three period types
(month / quarter / year) × five difficulty levels × 24 inconsistency codes.

> **Status:** the EMNLP 2026 paper is being written in `paper/`. The previous
> COLM 2026 work on a simpler clean-input task (FinBalance v1) is archived
> under [`_archive/`](_archive/README.md).

## Quickstart

Install (uv recommended):

```bash
uv sync
```

Generate a small pilot dataset:

```bash
python scripts/generate_dataset.py \
  --output-dir data/pilot \
  --records 24 \
  --industries professional_services retail subscription_saas \
  --period-types month quarter year \
  --levels 1 2 3 \
  --seed 42
```

Regenerate the canonical paper datasets:

```bash
python scripts/generate_standard_datasets.py \
  --base-dir data \
  --records-per-combo 8 \
  --negative-controls-per-code 4 \
  --overwrite
```

Run an evaluation ablation with OpenRouter (requires `OPENROUTER_API_KEY` in
`.env`):

```bash
python -m finbalance evaluate-ablations \
  --dataset data/coverage/records.jsonl \
  --model google/gemini-3-flash-preview \
  --ablations prompt_baseline forced_ledger_verifier \
  --max-records 15 \
  --output-dir results/pilot
```

Analyze with paired bootstrap:

```bash
python -m finbalance analyze-ablations --results-dir results/pilot
```

Regenerate the CPA verification packets (writes to `human_verification/samples/`):

```bash
python human_verification/generate_samples.py
```

## Reproducing the headline experiments

The dataset, models, and ablation matrix used in the paper are documented in
`POINTS.md`. The canonical evaluation runs are:

- **Multi-model baseline** at n=143 stratified records on six models (gemini-3-flash, gpt-5-low, claude-haiku-4.5, grok-4.3, deepseek-v3.2, qwen-3-235b).
- **Full ablation matrix** (prompt × visibility × tool variants) on gemini-3-flash; targeted ablations on deepseek-v3.2, claude-haiku-4.5, qwen-3-235b.
- **Paired bootstrap** (n=5000) over six metrics, computed by `python -m finbalance analyze-ablations`.

See `paper/sections/04_experimental_setup.tex` (work in progress) for the
final configuration.
