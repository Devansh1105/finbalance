\section*{Ethics Statement}

\benchmark{} should be used as an evaluation and stress-test tool, not
as a deployment certification for autonomous accounting.
Several systems produce well-structured outputs while silently violating
the accounting equation, a failure mode that could evade casual human
inspection.
The benchmark uses synthetic problems rather than real company data,
which avoids exposing private financial records while preserving the
core bookkeeping structure of the task, but it does not eliminate
deployment risk.
Any practical accounting workflow should therefore treat these results
as capability diagnostics and retain human oversight, auditable tools,
and domain-specific controls.
