\section{Introduction}

Recent financial NLP benchmarks have concentrated on question answering over existing reports, tables, or filings.
That setting is useful, but it bypasses a harder requirement that appears in real accounting workflows: synthesizing a correct financial statement from a stream of journal events.
Balance-sheet construction is a sharp stress test for structured reasoning because the model must simultaneously preserve sign conventions, accumulate account balances, apply period-end adjustments, and satisfy a global conservation constraint.

\benchmark{} targets this generative setting directly.
Each problem starts from opening balances, presents a sequence of double-entry journal events, optionally adds end-of-period adjustments, and asks the model to emit a JSON balance sheet with \texttt{assets}, \texttt{liabilities}, and \texttt{equity}.
The benchmark is built around a deterministic ledger engine, so every instance has exact ground truth by construction.
This lets us separate surface-level formatting issues from genuine failures in arithmetic, classification, account coverage, and closing-entry behavior.

Three research questions organize the paper:
\begin{enumerate}[leftmargin=*,topsep=2pt,itemsep=2pt,label=\textbf{RQ\arabic*:}]
\item How does balance-sheet generation accuracy degrade as accounting mechanisms compound across five difficulty levels?
\item When prompt strategy changes from \zs{} to \CoT{}, does performance improve uniformly, or only for specific model families and failure modes?
\item Which accounts and transaction factors dominate benchmark difficulty once per-problem outputs are retained for secondary analysis?
\end{enumerate}

\paragraph{Contributions.}
This paper makes four concrete contributions.
First, it introduces a deterministic benchmark for balance-sheet construction rather than document-centric financial question answering.
Second, it reports a six-model, two-prompt comparison on the released held-out split.
Third, it identifies account-level and factor-level failure patterns, including closing-entry blindness and arithmetic collapse under high structural recall.
Fourth, it shows how retained per-problem outputs support analyses that go beyond aggregate leaderboard scores.
